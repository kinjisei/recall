## Заход 8 — Полировка экранов

```

1. «Учёба»: строки «Задания от преподавателя» и «AI-квесты» приходят из async-запросов позже остальных и вываливаются вразнобой без анимации. Сделай: собери все async-источники хаба в Promise.all → рендери ВЕСЬ список строк разом со stagger-анимацией (animate-fade-up с задержками по индексу, как на Главной); пока грузится — лёгкие скелетоны той же высоты (без прыжков вёрстки).

3. AI-квесты (/quests): экран открывается «не сверху», нет заголовка и кнопки «Назад». Критерии: build; node scripts/ux-audit.mjs без новых замечаний; скриншоты «Учёбы» с заданиями (создать тестовую пару учитель+ученица с заданием и квестом) и /quests — посмотреть глазами. Коммит без пуша.
```

## Заход 9 — Игры: умные обманки + Спринт

```
Роль: ты ведущий разработчик Recall (CLAUDE.md). Жалоба владельца: в играх слов повторы («босс» дважды, при новом запуске те же слова) и ОЧЕВИДНЫЕ ответы (незнакомое слово угадывается исключением).
1. lib/wordPool.ts: новый экспорт pickDistractors(correct, pool, n): обманки ранжируются по похожести на правильный ответ — та же тема (topic), тот же уровень, та же часть речи (эвристика по окончанию русского перевода: -ть=глагол, -ый/-ой/-ий=прилагательное, иначе существительное), близкая длина перевода (±40%). Верный ответ не должен быть самым длинным/коротким в четвёрке — если выделяется, замени обманки.
2. Подключи pickDistractors во ВСЕ режимы с вариантами: Перевод, Значения (Match), Пропуск, Аудирование (QuizModes/MatchMode — сейчас обманки случайные).
3. SprintMode подключи к анти-повтору (withoutRecent/recordShown из wordPool) — в прошлый раз он был сознательно пропущен, это ошибка.
4. Проверь, что анти-повтор реально работает во всех режимах подряд: мини-тест на pickWords+recordShown (node, стрип типов) — два раунда подряд не пересекаются, пока пул позволяет.
Критерии: build; тест; ручная проверка: 3 раунда «Перевода» подряд — слова не повторяются, обманки одной части речи. Коммит без пуша.
```
