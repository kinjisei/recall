## Как этим вообще пользоваться

**Скилл — это пакет инструкций, который Claude подгружает под конкретную задачу.**
Есть два способа вызвать:

1. **Слэш-команда** `/имя-скилла текст задачи` — принудительный вызов. Используй,
   когда точно знаешь, какой скилл нужен («/grilling мой план запуска», «/pricing
   пересмотри тарифы»).
2. **Просто описать задачу** — Claude сам подхватит подходящий скилл по её смыслу
   («сделай лендинг для репетиторов» → сам возьмёт copywriting + cro). **Это
   основной способ.** Слэш нужен, только если Claude не догадался или ты хочешь
   конкретный подход.

**Что давать на вход:** скиллы не волшебники — им нужен контекст. Правило: дай
то, что дал бы живому специалисту (ссылку/файл/текст + цель + ограничения).
Примеры вводных — в таблицах ниже.

**На какой базе работают:** почти все — чистые «книги знаний» (инструкции + чек-листы),
работают везде. Исключения с внешними зависимостями помечены в таблицах:
Playwright (браузер), Python-библиотеки (pypdf, openpyxl), API генерации картинок.

## Практические правила

1. **Не зови скилл ради скилла.** «Почини баг» не требует скилла — скиллы дают
   плюс там, где нужна экспертиза (дизайн, продажи, право) или инструмент (PDF).
2. **Один скилл — одна задача.** Не проси «примени все маркетинговые скиллы» —
   получится каша. Claude сам скомбинирует 2-3, когда уместно.
3. **Чем конкретнее вводные, тем лучше выход.** «Напиши пост» → мусор.
   «Пост в TG-канал Recall для репетиторов, боль — ученики забывают слова,
   CTA — попробовать 14 дней» → рабочий текст.
4. **Скиллы читают файлы проекта.** Можно говорить «по нашему прайсингу из
   docs/monetization-draft.md» — скилл возьмёт данные оттуда.
